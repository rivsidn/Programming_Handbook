## 宏定义





› elisp 宏是什么，简单举例说明


• 我用“函数做计算，宏改写代码”这个角度来解释，再给你一个最小可运行的例子，不会一下子堆太多术语。

• 可以先把 Elisp 宏理解成一句话：

  宏不是先拿到“值”再运行，而是先拿到“代码”，把代码改写成另一段代码，再去执行。

  所以：

  - 函数：处理的是“参数的值”
  - 宏：处理的是“参数的原始表达式”

  ## 1. 先看函数

  例如：

  (defun my-add (a b)
    (+ a b))

  调用：

  (my-add 1 2)

  这里 1 和 2 会先作为参数传进去，然后函数计算结果。

  ## 2. 宏是什么

  宏更像“代码生成器”或“语法扩展器”。

  例如 Elisp 里很常见的 when，它其实就是宏：

  (when t
    (message "hello")
    (message "world"))

  它的意思是：

  > 当条件为真时，顺序执行后面的代码

  你可以把它大致理解成会被展开成类似这样：

  (if t
      (progn
        (message "hello")
        (message "world")))

  也就是说，when 不是普通函数，而是先把你的代码改写成 if + progn 这种更基础的形式。

  ## 3. 为什么需要宏

  因为很多“看起来像语言语法”的东西，用普通函数做不到。

  比如 if、when 这种结构有一个特点：

  它们不能把所有参数都先求值。

  如果 when 是普通函数，那么它的所有参数会先被求值，这就不对了。
  而宏可以决定：

  - 哪些部分先不求值
  - 怎么重组成新的代码
  - 最后再执行

  ## 4. 一个最简单的自定义宏例子

  (defmacro my-when (cond &rest body)
    `(if ,cond
         (progn ,@body)))

  用法：

  (my-when t
    (message "A")
    (message "B"))

  它展开后大致就是：

  (if t
      (progn
        (message "A")
        (message "B")))

  ## 5. 这里几个符号先简单认识一下

  在宏里你会经常看到反引号和逗号：

  - `：模板
  - ,x：把 x 的值插进去
  - ,@xs：把列表里的元素展开插进去

  所以这段：

  `(if ,cond
       (progn ,@body))

  意思就是：

  > 生成一个列表，形状是 (if ... (progn ...))，
  > 其中 cond 放到条件位置，body 里的多条代码展开到 progn 里面

  ## 6. 你现在先记住宏的本质

  最重要的是这句：

  宏接收的是代码，返回的也是代码。

  而函数通常是：

  函数接收的是值，返回的也是值。

  ## 7. 一个直观对比

  函数：

  (defun twice (x)
    (+ x x))

  宏：

  (defmacro my-print (x)
    `(message "值是: %S" ,x))

  调用：

  (my-print (+ 1 2))

  它会先展开成：

  (message "值是: %S" (+ 1 2))

  然后再执行。

 **意思就是不会想将参数计算成结果，而是直接传递代码。**

  ## 8. 初学阶段怎么理解就够了

  你现在不用急着自己写复杂宏，先记住：

  - when、unless、cond 这类“像语法”的东西里，很多都和宏有关
  - 宏的作用是让你定义“新的代码写法”
  - 宏本质上是在“生成代码”

    它们不能把所有参数都先求值。

    如果 when 是普通函数，那么它的所有参数会先被求值，这就不对了。
    而宏可以决定：

    - 哪些部分先不求值

  这一段如何理解呢？


• 我用“函数调用时参数先求值”这个规则来拆开讲，再对比 when 和 if 为什么不能写成普通函数。

• 关键在于：Elisp 调用普通函数时，通常会先把所有参数都算出值，再把这些值传给函数。

  先看一个普通函数：

  (defun my-func (a b)
    (list a b))

  调用：

  (my-func (+ 1 2) (+ 3 4))

  Elisp 会先做这两步：

  1. 先算 (+ 1 2)，得到 3
  2. 再算 (+ 3 4)，得到 7

  然后才真正调用：

  (my-func 3 7)

  所以普通函数拿到的是“已经算好的值”。

  问题就出在 if / when 这种结构上。

  比如：

  (if t
      (message "yes")
    (message "no"))

  你期望的是：

  - 条件是 t
  - 只执行 (message "yes")
  - 不执行 (message "no")

  但如果 if 是普通函数，会发生什么？

  Elisp 会先把所有参数都求值：

  1. 求值 t
  2. 求值 (message "yes")
  3. 求值 (message "no")

  这样一来，yes 和 no 都执行了，这显然不对。

  所以 if 不能是普通函数。
  它必须拥有“不要先把所有参数都求值”的能力。

  这就是你那段话里“它们不能把所有参数都先求值”的意思。

  when 也是同样道理。

  (when t
    (message "A")
    (message "B"))

  你希望的是：

  - 先看条件 t
  - 只有条件为真时，才去执行后面的 body

  如果 when 是普通函数，那么它后面的：

  (message "A")
  (message "B")

  会在调用前就被求值执行，那就失去了“按条件决定是否执行”的意义。

  所以 when 也不能设计成普通函数。

  那宏是怎么解决的？

  宏不会先拿到“参数的值”，而是先拿到“参数对应的代码”：

  (when t
    (message "A")
    (message "B"))

  when 宏拿到的大致是这些原始代码片段：

  - 条件代码：t
  - 主体代码：((message "A") (message "B"))

  然后它把这段代码改写成：

  (if t
      (progn
        (message "A")
        (message "B")))

  再由 Elisp 去执行这段新代码。

  所以宏的关键能力是：

  - 先不要急着算参数
  - 先把参数当“代码”看
  - 重新拼成另一段代码
  - 再执行

  可以把区别压缩成这两句：

  - 普通函数：先算参数，再调用函数
  - 宏：先改写代码，再执行改写后的代码

  再给你一个最直观的错误对比。

  假设你想写一个“条件执行”的普通函数：

  (defun my-when-bad (cond body)
    (if cond body nil))

  然后这样调用：

  (my-when-bad nil (message "hello"))

  问题是：在调用 my-when-bad 之前，(message "hello") 就已经被求值执行了。
  也就是说，即使 cond 是 nil，hello 还是会输出。

  这就说明：“条件决定是否执行某段代码”这类东西，不能靠普通函数直接实现。
